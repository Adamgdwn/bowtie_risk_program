## 2) Tester Script (Step-by-step)

1. Log in and confirm dashboard clarity.
2. Create a new bowtie with a realistic Top Event.
3. Build at least 3 threats, 3 consequences, and 4 barriers.
4. Add at least 1 escalation factor + 1 escalation factor control.
5. Use Worksheet steps and add notes in Step 1, 2, and 4.
6. Run at least 2 AI actions and insert selected suggestions.
7. Edit/reword a top event midway and confirm labels update correctly.
8. Test delete, undo/redo, copy/paste, and quick-add (`+`) behavior.
9. Export PNG and PDF using Canvas, Worksheet, and Both.
10. Share final rating + issues.

## 5) Triage Rules

1. `S1`: data loss, blocked core workflow, login failure.
2. `S2`: incorrect bowtie logic, broken AI insertion, export failure.
3. `S3`: UI confusion, layout issues, non-blocking bugs.
4. `S4`: polish/copy/minor visual issues.
5. Fix order: `S1 -> S2 -> top recurring S3`.

## 6) Rollout Plan

1. Wave 1 (3-5 testers): trusted users, 2 days.
2. Wave 2 (8-12 testers): mixed roles, 4-5 days.
3. Daily triage and hotfix pass.
4. Exit criteria: no S1, no open S2 in core flow, average usability >= 4/5.
